# Git

 - .git 폴더가 위치한 곳의 모든 내용을 관리해주는 프로그램 (윈도우즈에서 초기에는 숨겨져있다. 앞에 .이붙어서)
 - .git 폴더를 추가해 관리를 시작한 곳을 레포지토리(Repository)라고 부른다
 - .git 폴더에는 해당 레포지토리의 모든 정보가 기록된다 (변경 이력, 복원 지점, 버전 등..)
 - .git 내부의 정보를 이용해 해당 레포지토리를 예전 모습으로 되돌릴 수 있다
 - Git은 레포지토리 내부의 아주 작은 변화도 알아채고 관리해준다
 - Eclipse에 내장된 Git을 사용할 수도 있다

# Eclipse에서 Git관리 메뉴 열기 
 * window -> show view -> other -> Git (staging, repositories 등)

# 새 프로젝트로 레포지토리 만들기
- 새 프로젝트로 생성 -> 우클릭 -> Team -> Share Project -> 폴더위치 선택 후 레포지토리로 변환
- 원래 java-workspace에 있던 프로젝트가 레포지토리로 이동되어 .git의 관리를 받기 시작한다

# 작업물에 우클릭 -> team -> show in history / show in repositories
- history에서 head가 있는 부분이 현재 상태이다.
- 다른 커밋에서 우클릭 -> check out 하면 head가 옮겨간다.
